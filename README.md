# AI Fitness Tracker - Foundation Architecture

## Overview
This is a modern, AI-native fitness tracking application built with React, Vite, and individual modular layers for AI and State management.

## Project Structure

### 📂 `src/types/`
Defines the core data models of the application. 
- `Goal`: Tracking targets (weight, strength).
- `Entry`: Historical data (workouts, daily weight).
- `AIAnalysis`: Insights generated by the AI engine.

### 📂 `src/store/` (State Architecture)
Powered by **Zustand**. 
- **Centralized Store**: All goals, workouts, and weight entries are managed in a single state tree.
- **Persistence**: Automatically synced with `localStorage` via the `persist` middleware.

### 📂 `src/services/` (AI & Storage)
- **AIService**: Orchestrates communication with OpenAI API (GPT-4o-mini). It handles prompting logic, result interpretation, and generates progress forecasts.
- **StorageService**: Low-level abstraction for `localStorage` (backup/manual operations).

### 📂 `src/features/` (UI Modules)
The application is divided into feature-based blocks:
- `dashboard/`: Unified view showing overview metrics and AI-driven CTA buttons.
- `goals/`: CRUD operations for fitness objectives.
- `entries/`: Logging workflows for workouts and biometric data.
- `analytics/`: Visualization layer using **Recharts**.
- `ai/`: Dedicated views for deep AI breakdowns and recommendations.

## UI & UX Design
- **Theme**: Premium Dark Fitness.
- **Visuals**: Glassmorphism, Neon (#DFFF00) accents, high-density dashboard cards.
- **Typography**: `Outfit` for display (headings) and `Inter` for data.
- **Language**: 100% Russian UI (`src/constants.ts`).

## Component Interaction Flow
1. **User Action**: User logs a workout or weight change.
2. **State Transition**: Zustand store updates, triggering a local storage sync.
3. **Reactive Analysis**: Hooks detect state change and periodically trigger `AIService.analyzeProgress`.
4. **AI Generation**: OpenAI processes historical trends and deterministic analytics to return structured Russian recommendations.
5. **UI Update**: Dashboard cards reflect new forecasts and progress percentage.

## Deployment
Configured for **Vercel** compatibility with standard standard Vite build outputs.
